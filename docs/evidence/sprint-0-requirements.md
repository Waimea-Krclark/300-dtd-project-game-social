# Project Requirements

## Identified Problem or Need

There isn't one centralised way to get news and information on games from developers, discuss and share about those games, as well as those developers being able to communicate with and get feedback from its players.
A system that provides a place for discussions about games, as well as announcements, patch notes and other community events to be announced by developers would be useful for both developers and gamers.

## End-User Requirements

There would be two kinds of typical end users for this project.
One kind of end-user would be consumers of games who are potentially into the game development process, or just interested and passionate about games in general. They would have a wide range of knowledge about different games, and would likely have information, opinions and insight to share to or gather from other users.

Another kind would be the developers, who would need a clear and well designed way to communicate and share with the players. They should be able to create game related announcements and posts as well as general community posts. Developers would need a space for their community, such as a dedicated thread/page for their game.

To cater for both these kinds of end-users, an intuitive and robust system that allows both kinds of users to interact and achieve their needs within the application should be created.

## Proposed Solution

To create an application for both players and developers, the system will have the following features:
- Users accounts (standard and developer accounts)
- Developers can create a location for their game (A page with details, images, trailers ect)
- Developers can make posts for the games here for things such as updates, announcements and community events.
- Players can browse these game pages and posts
- Users can reply to other posts/messages
- Game pages will have a discussion section where any user can make posts (for sharing, giving information ect.)
- A home page with search functions that allow users to easily browse games and/or posts

# Relevant Implications

## ACCESSIBILITY

Accessibility is the implication about how the system is able to be used by anyone. The system should be accessible to all users, regardless of experience, disabilities, and skill. It should also be accessible in multiple ways, for example, it should work on mobile devices as end-users are likely to want to access the app anywhere, and a mobile-friendly UI means that the app would be easy to user for anyone on their phone.

### Relevance to the System

Accessibility is important to this project, because of the wide range of potential user that may utilise the application. Any person could sign in to the app and use it, meaning the app needs to be as broadly accessible as possible, so it doesn't block some users from being able to use the app properly. The player side of things is the most important for this, as anything a standard non-developer user is able to do could be done by any person with any level of skill, knowledge or ability.

### Impact / Considerations

I will need to ensure that all parts of the system are accessible to a wide range of users. Doing things such as adding alt text to images, choosing colours that contrast well to increase readability and laying out the site in a clear easy to understand way will help some users with disabilties, and ensuring that the functions are simple and easy to understand will help users who may have less knowledge or skill. With considerations of accessiblity through all parts of the creation of this system, it should result in a broad system that can be comfortably used by many different users.
